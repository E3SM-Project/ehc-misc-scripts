# Scripts for modifying input datasets
--------------------------------------------------

## Updated ocean flux file used for transient simulation:
### Script name - `add_buffer_year.sh`

* The historical ocean flux file, `ne30pg2_CSEM_historical_ocean_flux_1849-2014_c20240225.nc`, lacked data for the December 2014 month (the data was for time 2014-12-15) and the run using this file was crashing with error:

```
set_wghts_indices cannot not find model time in: /lcrc/group/e3sm/public_html/inputdata/atm/cam/ggas/ne30pg2_CSEM_historical_ocean_flux_1849-2014_c20240225.nc
```

* `/lcrc/group/e3sm/data/inputdata/atm/cam/ggas/ne30pg2_CSEM_hist_ssp370_ocean_flux_1850-2100_c20231108.nc` file had December 2014 data but the data begins on 1850-01-15 instead of January 1st and therefore was giving the same error described above.
* To overcome this error, a new file is created by adding a buffer year for 2015 to the historical file - `ne30pg2_CSEM_historical_ocean_flux_1849-2014_c20240225.nc`. 
* The new file location is: `/lcrc/group/e3sm/data/inputdata/atm/cam/ggas/ne30pg2_CSEM_historical_ocean_flux_1849-2015_c20260128.nc`
* The new file has data from 1849-01-15 to 2015-12-15.

## Updated parameter file:
### Script name - `modify_paramfile.py`
* Script designed to modify and update E3SM Land Model (ELM) parameter file.
* Update two parameters (`soilpsi_off` and `kmax`) in the 25 pft parameter file `clm_params_c211124.nc`.
* The updated values for these two parameters are based on tuned paramter file generated by ELM BGC group - `clm_params_c230517_phs_50pfts_tuned.nc`.
* The updated parameter file name is : `clm_params_c211124_tuned.nc`.
